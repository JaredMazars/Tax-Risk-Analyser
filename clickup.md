1. Purpose of Board View

The Board View provides a visual, column-based workflow for managing tasks as they move through different stages of progress.
Each column represents a status or workflow stage, and each card represents a task.

2. Board Structure
Columns

Columns represent task statuses (e.g. To Do, In Progress, Review, Done)

Column order is customizable via drag-and-drop

Columns can be added, renamed, or removed

Each column can show:

Task count

Aggregated data (e.g. total time estimated, total points)

Columns can be collapsed or expanded

Cards (Tasks)

Each card represents a single task

Cards display configurable information such as:

Task detail

Status

Priority

Due date

Tags

Team members

Custom field values

Budgeted vs Actual Time

Card appearance updates dynamically as task data changes

3. Drag & Drop Interaction

Tasks can be dragged:

Between columns (status change)

Within a column (priority or manual ordering)

Dragging a task between columns automatically updates its status

Drag actions trigger:

Automations

Activity logs

Progress updates

Visual indicators show valid drop zones

4. Card Interaction

Clicking a card opens the full task detail view

Inline editing supported for for Admin users in the Subservicelinegroup and System Admin:

Task title

Team

Due date

Priority

Selected custom fields

Quick actions available directly on cards (e.g. assign, reschedule)

5. Filtering & Sorting

Board can be filtered by:

Team member

Priority

Tags

Due dates

Custom fields

Multiple filters can be applied simultaneously

Sorting options include:

Manual order

Due date

Priority

Custom field values

Filters and sorts apply across all columns

6. Grouping Options

Tasks can be grouped within the board by:

Team Member

Priority

Custom fields

Tags

Grouping creates sub-sections inside columns

Groups can be collapsed or expanded

7. Swimlanes

Horizontal swimlanes can be enabled to visually separate tasks by:

Assignee

Priority

Custom field

Due date ranges

Swimlanes improve clarity in high-volume boards


9. Column & Board Calculations

Columns can display roll-up metrics such as:

Total number of tasks

Sum of time Actual and Budget

Sum of custom numeric fields

Board-level summaries can show:

Total tasks

Completed vs remaining

Overall progress percentage



14. Visual Customization

Columns can have:

Color indicators

Icons or labels

Cards visually reflect:

Priority

Overdue status

Blocked dependencies

Compact or detailed card display modes available

15. Navigation & Usability

Horizontal scrolling for large boards

Keyboard shortcuts for navigation and actions

Zoom levels for dense or high-level viewing

Sticky column headers for clarity

Changes are reflected instantly without refreshing

Summary

The Board View functions as a dynamic, visual workflow engine that enables users to:

See work status at a glance

Move tasks through a process intuitively

Enforce workflow rules

Manage capacity and priorities visually

Collaborate in real time
